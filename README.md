## 04.03.26 - Чему я научился

CI/CD (Continuous Integration & Continuous Deployment):

Continuous Integration (CI):

Сборка: Автоматизация процесса создания Docker-образа из Dockerfile.

Тесты: Автоматический запуск скриптов для проверки работоспособности кода.

Линтинг: Автоматическая проверка кода на ошибки и «мусор» (например, дубликаты в app.py).

Continuous Delivery/Deployment (CD):

Доставка: Автоматическая отправка собранного образа в Registry (хранилище).

Развертывание: Автоматический доступ по SSH к серверу и выполнение команд деплоя (аналог ./run.sh) для обновления приложения без ручного участия.

Трансформация процесса (от ручного к DevOps):

Ручной режим: Ты сам пишешь код, сам запускаешь docker-compose up --build и сам проверяешь работоспособность через curl.

DevOps-режим: Ты делаешь git push, а конвейер (пайплайн) самостоятельно берет на себя:

Проверку Dockerfile и сборку образа.

Тестирование работоспособности (запуск временного контейнера и curl).

Развертывание на сервере при успехе или уведомление об ошибке.

## 03.03.26 - Чему я научился

Flask & Networking (Programming & Networking):

from flask import Flask: Это база. Импорт каркаса веб-сервера, без которого Python не поймет, как слушать сеть.

app = Flask(__name__): Создание объекта «приложение». Запуск «двигателя» — теперь есть живой процесс для обработки запросов.

@app.route('/'): Роутинг (маршрутизация). Декоратор, который говорит серверу выполнять функцию hello(), если кто-то (Nginx или curl) стучится на главную страницу.

app.run(host='0.0.0.0', port=5000): Ключевая строка для DevOps:

host='0.0.0.0': Слушать всех. Открывает «ворота» настежь, чтобы сосед по сети backend (Nginx) мог достучаться до сервиса.

port=5000: Номер «двери», в которую нужно стучать.

Docker & Infrastructure (Containers):

Зависимости в образе: Понял, что новые библиотеки (flask) нужно обязательно прописывать в Dockerfile через RUN pip install, иначе контейнер не запустится.

Сетевая изоляция: Разобрался, что внутри кастомной сети bridge (у нас это backend) контейнеры видят друг друга по именам сервисов.
