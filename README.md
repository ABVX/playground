CI/CD (Continuous Integration & Continuous Deployment):

Continuous Integration (CI):

Сборка: Автоматизация процесса создания Docker-образа из Dockerfile.

Тесты: Автоматический запуск скриптов для проверки работоспособности кода.

Доставка: Автоматическая отправка собранного образа в Registry (хранилище).

Развертывание: Автоматический доступ по SSH к серверу и выполнение команд деплоя (аналог ./run.sh) для обновления приложения без ручного участия.

Трансформация процесса (от ручного к DevOps):

Ручной режим: Ты сам пишешь код, сам запускаешь docker-compose up --build и сам проверяешь работоспособность через curl.

DevOps-режим: Ты делаешь git push, а конвейер (пайплайн) самостоятельно берет на себя:

Проверку Dockerfile и сборку образа.

Тестирование работоспособности (запуск временного контейнера и curl).

Развертывание на сервере при успехе или уведомление об ошибке.
